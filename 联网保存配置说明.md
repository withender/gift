# 礼物盲盒 · 联网保存配置（Cloudflare Pages + D1）

配置完成后，你和她打开**同一个带 `?room=xxx` 的链接**，礼物列表和抽奖结果会同步到云端。

---

## 一、创建 D1 数据库

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 左侧 **Workers & Pages** → **D1 SQL Database**
3. **Create database**，名称填 `gift-box`
4. 创建后复制 **Database ID**（一串 UUID）

---

## 二、初始化数据表

在 D1 数据库页面点 **Console**，粘贴 `schema.sql` 里的 SQL 执行：

```sql
CREATE TABLE IF NOT EXISTS gift_rooms (
  room_id TEXT PRIMARY KEY,
  items TEXT NOT NULL DEFAULT '[]',
  drawn TEXT NOT NULL DEFAULT '[]',
  updated_at INTEGER NOT NULL
);
```

或在本地（已安装 Node）：

```bash
cd deploy
npx wrangler d1 execute gift-box --remote --file=schema.sql
```

---

## 三、绑定 D1（重要：按钮是灰的时看这里）

若 Pages **设置 → 绑定 → 「+ 添加」是灰色**，并提示 **「绑定在通过 wrangler.toml 进行管理」**，说明**不能在网页里点添加**，必须改 `wrangler.toml`。

### 步骤 1：复制 Database ID

1. 左侧 **Storage & Databases** → **D1 SQL Database**
2. 点数据库 **`gift-box`**
3. 在概览页找到 **Database ID**（一串 UUID，如 `a1b2c3d4-e5f6-7890-abcd-ef1234567890`）
4. **复制**这串 ID

### 步骤 2：填进 wrangler.toml

打开 `deploy/wrangler.toml`，把：

```toml
database_id = "PASTE_YOUR_D1_DATABASE_ID_HERE"
```

改成（示例）：

```toml
database_id = "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
```

**注意：** 必须是真实 UUID，不能留占位符，否则部署会失败。

### 步骤 3：推送

```bash
cd deploy
git add wrangler.toml
git commit -m "bind D1 database"
git push github master
```

部署成功后，**绑定** 页面会自动显示 `DB` → `gift-box`，无需点「添加」。

---

### 备选：不想用 wrangler.toml

若希望能在 Dashboard 里点「添加」，可**删除** `wrangler.toml` 后 push，再在 **绑定** 里手动添加 D1。  
（当前项目已启用 wrangler 管理，**推荐用上面步骤 1～3**）

---

## 四、推送代码触发部署

```bash
cd deploy
git add .
git commit -m "add cloud sync with D1"
git push github master
```

等 Cloudflare **Deployments** 显示 Success。

## 五、验证是否成功

1. 用 **管理员链接**（带 `&admin=1`）打开 pages.dev，页面上方应显示 **「☁️ 已同步云端」** 和共享链接栏
2. 添加几件礼物，刷新页面，列表还在
3. 把**普通链接**（只有 `?room=xxx`，**不带** `admin=1`）发到另一台手机打开，应看到**同一份礼物池**，且**看不到**顶部共享链接

---

## 六、管理员模式（隐藏共享链接）

为避免她打开链接时看到「共享链接」、察觉你在同步查看礼物池，页面做了区分：

| 谁 | 链接格式 | 页面上看到什么 |
|----|----------|----------------|
| **你（管理）** | `https://xxx.pages.dev/?room=kjtyiun&admin=1` | 显示共享链接、云端状态、「复制」按钮 |
| **她（抽奖）** | `https://xxx.pages.dev/?room=kjtyiun` | **不显示**共享链接和云端状态，只有盲盒界面 |

**要点：**

- 共享链接**默认已隐藏**，她打开普通链接看不到。
- 只有链接带 **`&admin=1`** 时，才会显示共享链接和云端状态。
- 点「复制」得到的链接**不含** `admin=1`，可直接发给她。
- 你录入礼物、她抽奖，仍通过同一个 `?room=` 房间号在云端同步。

**示例：**

```
你自己用：  https://mygift-7df.pages.dev/?room=kjtyiun&admin=1
发给她：    https://mygift-7df.pages.dev/?room=kjtyiun
```

## 七、日常使用

| 操作 | 说明 |
|------|------|
| 自己管理礼物池 | 用带 **`&admin=1`** 的链接打开 |
| 复制链接发给她 | 在管理员模式下点 **「复制」**（或手动去掉 `&admin=1`） |
| 你录入礼物 | 她那边刷新或重新打开链接即可看到 |
| 她抽奖 | 你这边用管理员链接刷新，也能看到「已抽」记录 |
| 仅本机模式 | D1 未配置时仍可用 localStorage，不影响基本功能 |

## 常见问题

**显示「云端未配置」**  
→ Pages 项目未绑定 D1，或 binding 名称不是 `DB`

**API 404**  
→ 确认仓库里有 `functions/api/room/[id].js`，且已重新部署

**数据不同步**  
→ 确认两人链接里 `?room=` 后面的房间号**完全一致**

**她看到了共享链接**  
→ 检查发给她的链接是否误带了 `&admin=1`，应只保留 `?room=xxx`
