# 礼物盲盒 · 联网保存配置（Cloudflare Pages + D1）

配置完成后，你和她打开**同一个带 `?room=xxx` 的链接**，礼物列表和抽奖结果会同步到云端。

---

## 一、创建 D1 数据库

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com)
2. 左侧 **Workers & Pages** → **D1 SQL Database**
3. **Create database**，名称填 `gift-box`
4. 创建后复制 **Database ID**（一串 UUID）

---

## 二、初始化数据表

在 D1 数据库页面点 **Console**，粘贴 `schema.sql` 里的 SQL 执行：

```sql
CREATE TABLE IF NOT EXISTS gift_rooms (
  room_id TEXT PRIMARY KEY,
  items TEXT NOT NULL DEFAULT '[]',
  drawn TEXT NOT NULL DEFAULT '[]',
  updated_at INTEGER NOT NULL
);
```

或在本地（已安装 Node）：

```bash
cd deploy
npx wrangler d1 execute gift-box --remote --file=schema.sql
```

---

## 三、把 D1 绑定到 Pages 项目（在 Dashboard 操作，不要改 wrangler.toml）

> ⚠️ **不要在 `wrangler.toml` 里填假的 `database_id`**，会导致 Pages 部署失败。

1. **Workers & Pages** → 点你的 Pages 项目（如 `mygift`）
2. **Settings** → **Functions**
3. **D1 database bindings** → **Add binding**
   - Variable name：`DB`（必须叫 DB，与代码一致）
   - D1 database：选 `gift-box`
4. 保存后，到 **Deployments** → **Retry deployment** 或重新 push 代码

`wrangler.toml` 里只需保留 `name`、`compatibility_date`、`pages_build_output_dir`，**不要**写 `[[d1_databases]]` 占位符。

**可选：** 若要用 wrangler.toml 绑定 D1，必须把 `database_id` 换成 D1 控制台里的**真实 UUID**（格式如 `a1b2c3d4-e5f6-...`）。

---

## 四、推送代码触发部署

```bash
cd deploy
git add .
git commit -m "add cloud sync with D1"
git push github master
```

等 Cloudflare **Deployments** 显示 Success。

---

## 五、验证是否成功

1. 打开你的 pages.dev 链接，页面上应显示 **「☁️ 已同步云端」**（不是「云端未配置」）
2. 添加几件礼物，刷新页面，列表还在
3. 把链接（带 `?room=kkxxxxxx`）发到另一台手机打开，应看到**同一份礼物池**

---

## 六、日常使用

| 操作 | 说明 |
|------|------|
| 复制共享链接 | 点页面上方 **「复制」**，发微信给她 |
| 你录入礼物 | 她那边刷新或重新打开链接即可看到 |
| 她抽奖 | 你这边刷新也能看到「已抽」记录 |
| 仅本机模式 | D1 未配置时仍可用 localStorage，不影响基本功能 |

---

## 常见问题

**显示「云端未配置」**  
→ Pages 项目未绑定 D1，或 binding 名称不是 `DB`

**API 404**  
→ 确认仓库里有 `functions/api/room/[id].js`，且已重新部署

**数据不同步**  
→ 确认两人链接里 `?room=` 后面的房间号**完全一致**
